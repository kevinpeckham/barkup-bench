# Addendum brief — Study AH: memo saturation (the shipped memo at its cap)

**Pre-registration, committed before any scored AH run.** The
session-notes memo is the most load-bearing mechanism the series has
shipped (T/V/W/Y/AB), and every measurement of it used 3–6 notes.
The shipped implementation (`src/shipped/session-notes.ts`, verbatim
replicator ports) has a hard cap the studies never touched:
`MAX_SESSION_NOTES = 20`, enforced by `normalizeSessionNotes`, which
**silently drops** everything past twenty — it never throws, and
nothing in the prompt rule mentions the cap (only the tool schema's
"max 20 notes" phrase discloses it). Study AH measures the memo at
scale: recall and rule-application with a full memo, full-replace
integrity as the list grows, and the unspecified behavior at the cap
edge — the series' recurring villain (silent loss) possibly designed
into its own favorite mechanism.

## Corpus

`corpus/memo-scale.json`, **seed 20260718**, generated by
`scripts/generate-memoscale-corpus.ts` from the size-extension trees
(constructions in `src/corpus/memoscale.ts`, validators unit-tested
before any scored call). Every note carries a unique registered
needle token, so presence/absence in memos, tool arguments, and
edits is deterministic. The memo renders through the SHIPPED
`formatSessionNotesBlockV2` (PRECEDENCE clause included), and the
integrity arm uses the shipped prompt rule, tool description, and
`normalizeSessionNotes` clamp — Study W's verbatim ports, unchanged.

## Arms (3)

- **AH-recall** — single-turn edits against a minimal view, with a
  memo of N notes (N ∈ {5, 20}); the instruction references one fact
  note's content without stating it ("rename … to the launch
  codename") in Study T's callback style. The needed note sits at a
  registered position (first / middle / last within the rendered
  block); all other notes are same-schema distractors with disjoint
  needles. Graded against the expected tree (the needle applied
  exactly). 5 tasks × 3 positions × 2 N = 30 cells per model.
- **AH-rules** — same construction, but the memo's needed note is a
  standing RULE covering exactly the requested edit (unit-validated:
  one and only one rule applies), which the model must apply
  unprompted alongside the requested change. Contamination scan:
  any OTHER note's needle appearing in the final tree, Study Z
  style. 30 cells per model.
- **AH-integrity** — the agent side: system carries the shipped
  prompt rule + the `update_session_notes` tool + a memo preloaded
  with K notes (K ∈ {10, 19, 20}); the user declares ONE new fact
  (registered formulaic template — Y already validated casual
  phrasing) and requests a simple edit. Expected: one tool call with
  the COMPLETE updated list, plus the patch. Grading on the RAW tool
  argument (before the clamp) and the normalized result: old-needle
  preservation, new-needle presence, and at K=20 (where the update
  cannot fit) a registered behavior taxonomy: pruned-an-old-note /
  sent-21-and-lost-newest-to-the-clamp /
  sent-21-and-lost-an-old-note / declined-or-asked / skipped-tool.
  10 scenarios × 3 K = 30 cells per model.

**Models:** sonnet-4.5, gemini-3.5-flash, opus-4.8. 270 cells.
Protocol: temperature 0, minimal views (conditionF + VIEW_RULES),
shipped applier, ≤3 correction rounds, resumable JSONL
`results/raw/studyah-<model>.jsonl`, cache audit re-run.

## Pre-registered hypotheses and gates

- **AH-H1 (recall at the cap):** per model, N=20 recall ≥ 13/15 AND
  not significantly below N=5 (McNemar paired by task template).
  Anchors: T-notes 80/80 and W 36/36 at small N; Z found no burial
  in ~3.3k-token packs.
- **AH-H2 (rules at the cap):** per model, N=20 unprompted rule
  application ≥ 13/15, AND zero contamination events across all 90
  rule cells (hard zero — 1,080 prior conflict/standing cells never
  produced one).
- **AH-H3 (full-replace integrity where the update fits):** per
  model, at K=10 AND at K=19: the tool is called and the returned
  list contains ALL old needles plus the new one in ≥ 9/10
  scenarios. (Anchor: W saw one drop event in 72 sessions at small
  N; K=19 → 20 notes is the largest fitting update the shipped cap
  allows.)
- **AH-H4 (the cap edge — DESCRIPTIVE, no gate):** the K=20
  taxonomy, per model, plus post-clamp loss accounting (which note
  actually survives the shipped `normalizeSessionNotes` given what
  the agent sent). First data on unspecified behavior; expectations
  deliberately unregistered.
- **The study gate: AH-H1, AH-H2, and AH-H3 pass on all three
  models.**

## Interpretation table (pre-registered)

| Outcome | Reading |
|---|---|
| All pass, H4 benign (agents prune deliberately) | The memo is validated to its shipped cap; the cap edge is survivable; document the pruning behavior as the de facto policy |
| All pass, H4 lossy (clamp eats notes silently) | The mechanism holds but the CAP is a shipped footgun: the fix is app-side (surface the cap, or evict-oldest deterministically) — filed for the replicator with the measured loss mode |
| H1/H2 fail (burial inside the memo) | The memo has a size ceiling below its cap — shipped guidance changes from "cap 20" to the measured ceiling, and note-count becomes a monitored budget |
| H3 fails (rewrite fidelity decays with length) | Full-replace is the wrong protocol at scale — delta-update or app-side merge becomes the recommended mechanism, measured before shipping |

## Protocol notes, registered up front

- The integrity arm's tool execute returns the normalized list to
  the agent (shipped behavior) — an agent that notices the echo and
  reacts is measured as whatever its final tool state shows.
- The tool schema's "max 20 notes" phrase is part of Study W's
  verbatim port and is the ONLY cap disclosure the agent gets,
  exactly as shipped.
- AH-recall/AH-rules isolate READ-side scale (app-authored memos,
  no tool); AH-integrity isolates WRITE-side scale. No arm mixes
  them, so a failure localizes.
- Needle grading is exact-substring on registered unique tokens;
  rephrasing a note while preserving its needle counts as
  preserved (disclosed: text fidelity beyond the needle is not
  graded).

**Expected spend $5–9** (270 cells, ~4k input tokens each); abort
past $25. Honesty rules unchanged: this brief, the seed, the note
templates, the taxonomy, and all thresholds are committed before
the first scored call; results publish whatever they show.
